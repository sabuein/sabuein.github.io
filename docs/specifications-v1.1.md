# Specifications v1.1

## Project
Personal website redesign for `sabuein.github.io`

## Status
Planning only — no implementation yet.

## Purpose of v1.1
This specification refines the v1 architecture by defining the content strategy, page messaging, UX copy, contact actions, identity wording, and how key credentials such as Ghost Expert and AI-assisted workflows should be presented.

---

## 1. Content strategy summary

The site should remain intentionally simple and split across three clear destinations:

1. **Home page**  
   A concise landing page that introduces Salaheddin AbuEin, highlights his strengths and credentials, and directs visitors to the appropriate destination.

2. **Resume page**  
   A standalone, recruiter-friendly CV/resume page with improved writing and stronger positioning.

3. **Portfolio redirect page**  
   A branded redirect page that sends visitors to the main portfolio on `abuein.dev`.

The home page should not duplicate the full resume or full portfolio. It should act as a clear, well-designed entry point.

---

## 2. Home page content goals

The home page should communicate the following quickly:

- who Salaheddin AbuEin is
- what he does
- what makes him distinctive
- where to go next
- how to contact him

### Key identity points
- Full-stack web developer
- Official Ghost Expert
- 10+ years of experience

Ghost Expert is an important differentiator and should be featured primarily on the home page.

### Recommended usage
- include in hero role line
- include a small credential badge
- optionally link to the official Ghost Expert profile
- mention Ghost naturally in resume experience bullets, but do not over-repeat the credential there

### Goal
Position Salaheddin as someone credible for:
- Ghost freelance work
- CMS projects
- broader web development work

## 6. Resume page content strategy

The resume page should be more concise and recruiter-focused than the current version, while preserving depth and credibility.

### Resume page should include
- strong summary
- technical skills
- employment history
- education
- optional certifications
- PDF download
- print-ready formatting

### Resume page should avoid
- excessive decorative text
- repeated claims already obvious from work history
- too many long bullets
- over-explaining older non-technical roles

The current role at Rapid Information Systems should show:
- progression
- technical breadth
- ownership
- platform work

- practical AI use

- Lead development and ongoing improvement across multiple production websites supporting Rapid Information Systems and associated businesses.

## 9. AI wording strategy

The site should present AI usage as practical engineering support, not hype.

### Approved AI use themes
- debugging
- feature development
- code review assistance
- UI/style refinement
- issue resolution
- technical research
- written communication
- documentation support

### Avoid
- vague claims about “AI innovation”
- exaggerated automation claims
- wording that weakens technical credibility

### Recommendation
Keep AI mention:
- in the resume current role
- possibly lightly echoed in the home summary if needed
- not made into a major hero slogan

## 12. Contact strategy

The site should make it easy for people to contact Salaheddin in the way they prefer.

### Primary contact action
- Email

### Secondary contact actions
- WhatsApp
- LinkedIn
- GitHub
- Call
- Download vCard
- Share

### Recommendation
Email remains the most universal and professional CTA, while WhatsApp and call are useful especially on mobile.

---

## 13. vCard strategy

Include a downloadable `.vcf` file.

### Benefits
- useful on mobile
- quick save-to-contacts
- good complement to tel/mail links
- low maintenance

### Recommendation
Add a simple `Download Contact Card` action on the home page and possibly the resume page.

---

## 14. Share strategy

Use the Web Share API where supported.

### Suggested share use cases
- share home page
- share resume page
- share portfolio destination

### Fallback
If Web Share API is unavailable:
- copy URL to clipboard
- show small success feedback message

---

## 15. Resume PDF strategy

The resume page should support both:
- direct print-to-PDF
- downloadable static PDF

### Recommendation
Include a visible action:
- `Download PDF`

And ensure the PDF is also cached by the service worker for offline availability.

---

## 16. Keffiyeh-inspired visual strategy

The keffiyeh motif is meaningful and should be used deliberately, not as decoration alone.

### Approved use
- subtle hero texture
- divider bands
- footer ornament
- small accent surfaces

### Resume page
Use it very lightly so the page remains professional and print-friendly.

### Home page
Use it subtly to moderately.

### Redirect page
Can use it a little more strongly, since the page is short and visual.
